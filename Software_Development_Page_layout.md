# REGTEK CONSULTING - SOFTWARE DEVELOPMENT PAGE STRUCTURE & BUILD GUIDE

## Overview

This document defines the complete structure, layout, and purpose of each section for the Regtek Consulting Software Development service page. Use this as the foundation when building this page in Breakdance.

**Service positioning:** Custom apps, internal dashboards, and workflow automation, with healthcare as the primary target market and small businesses as a secondary audience.

**Page goal:** Same as homepage. Book free consultation as primary conversion, newsletter signup as secondary.

- **Colors:** Refer to "Mood Board" document in project knowledge

## CONVERSION OPTIMIZATION NOTES

### CTA Strategy: MODERATE (4 Touchpoints)
1. Hero section (primary position)
2. After case examples ("Want results like these?")
3. After "How It Works" ("Start With Step 1")
4. Final CTA section

### Section-Specific Conversion Notes
- Hero must communicate value in under 5 seconds
- Case examples are the heaviest converter, place them mid-page after value pillars
- FAQ handles cost objections before final CTA, critical for higher-ticket service
- Founder section builds trust at the right moment, after they've seen the work but before final ask

---

## PAGE-SPECIFIC POSITIONING NOTES

### Voice Reminders
- Operational language, not technical jargon
- Speak to the business owner or operations leader, not the IT person
- Specific examples beat vague claims every time
- Confidence without arrogance

### What This Page Should NOT Do
- Should not list programming languages or technical stack
- Should not discuss development methodology in detail
- Should not lead with the tooling or process used to build
- Should not feel like a typical "web development agency" page

### Differentiation Anchors
- Operations background (Inc 5000)
- Healthcare specialty
- Speed of delivery
- Founder-led, not handed off to junior devs
- Code ownership and no lock-in
